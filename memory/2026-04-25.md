## Critical findings
1. **`effect_update_depth_exceeded` ROOT CAUSE**: `(admin)/+layout.svelte:37` — `$effect` reads AND writes `lastStatus` ($state). Svelte 5 registers it as dependency → infinite loop. Fix: `untrack()`.
2. **XSS**: `{@html dividerTitle}` in `(store)/+page.svelte:248` renders raw HTML from Firebase.

## Key lesson
In Svelte 5, if a `$effect` reads a `$state` variable (even in a conditional) and then writes to it, you get an infinite loop. Always use `untrack()` for reads that shouldn't trigger re-runs.
