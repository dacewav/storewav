04 — Decisiones Clave
Elegí tu camino
Hay varias decisiones que definen la dirección del proyecto. Clickeá la opción que preferís — se guarda automáticamente.
¿Dónde hosteás el audio?
Cloudflare R2 (gratis 10GB, CDN global)
Firebase Storage (ya lo usás, fácil)
Supabase Storage (gratis 1GB, PostgreSQL)
¿Procesador de pagos?
Stripe (estándar industria, webhooks)
Mercado Pago (si vendés en LATAM)
Ambos (máximo reach, más laburo)
¿Sistema de auth?
Firebase Auth (integrado, Google/email)
Clerk (UI prearmada, fácil)
Lucia Auth (lightweight, self-hosted)
¿Base de datos principal?
Firebase RTDB (ya la tenés, realtime)
Firestore (queries más potentes)
Ambas (RTDB realtime + Firestore queries)
¿Emails transaccionales?
Resend (3K gratis/mes, simple)
Postmark Ninguno por ahora (agregar después)
¿Analytics?
Plausible (privado, lightweight)
Umami (self-hosted, gratis)
Firebase Analytics (ya integrado)
05 — Roadmap
Fases de desarrollo
Cada fase es un deploy funcional. No pasás a la siguiente hasta que la anterior está sólida. Clickeá para expandir los detalles.
Fase 0 — Fundación ~1 semana
Scaffold + migrar diseño existente
Levantar SvelteKit, replicar la estética del catalog actual (tema oscuro, Syne, DM Mono, efectos), y tener el player básico funcionando con tus beats actuales.
Setup
  • Init SvelteKit + adapter-cloudflare
  • Configurar Firebase client (migrar config.js)
  • Setup TypeScript + strict mode
  • Wrangler config para Cloudflare Pages
  • CI/CD: push → deploy automático
Diseño
  • Layout global con nav + player sticky
  • Sistema de temas (dark/light) con stores
  • Tipografía: Syne + DM Mono (migrar)
  • Paleta CSS variables (migrar desde catalog)
  • Componente BeatCard.svelte
  • Grid responsive de beats
Audio
  • Migrar player.js → stores/player.ts
  • Waveform component (SVG reactivo)
  • Reproductor con controles: play/pause, seek, volume
  • Preview con tag de audio (30s o loop)
SvelteKit 2 Svelte 5 TypeScript Firebase SDK Cloudflare Pages Vite
Fase 1 — Core Features ~2 semanas
Tienda completa + filtros + SEO
Migrar todas las features del catalog: filtros, búsqueda, tag cloud, wishlist, modal de beat. Agregar SSR para que cada beat sea indexable por Google.
Tienda
  • Migrar filters.js → FilterBar.svelte
  • Search con debounce (migrar utils)
  • Tag cloud interactivo
  • Modal de beat con licensing info
  • Wishlist con persistencia (localStorage + auth sync)
  • Export wishlist por WhatsApp (migrar)
SEO & Performance
  • SSR: página /beat/[id] con meta tags
  • Open Graph tags (imagen + audio preview)
  • Sitemap.xml dinámico
  • Structured data (JSON-LD para MusicRecording)
  • Lazy loading de audio y waveform
Efectos (migrar con cuidado)
  • Cursor glow (solo desktop, respetar prefers-reduced-motion)
  • Card tilt effect
  • Scroll-triggered animations (Intersection Observer)
  • Stagger reveal en grid
SvelteKit SSR Framer Motion OG Image generation
Fase 2 — Commerce ~2-3 semanas
Auth + pagos + licensing
La fase que convierte el catálogo en negocio. Auth para trackear compras, Stripe para cobrar, Cloud Functions para procesar pagos y generar links de descarga.
Autenticación
  • Firebase Auth setup (Google + Email)
  • Login/Register UI components
  • Auth store reactivo
  • Route guards (admin protegido)
  • Página /account con historial de compras
Stripe Integration
  • Stripe account + productos/precios configurados
  • LicenseSelector component (4 tiers)
  • Stripe Checkout Session (Cloud Function)
  • Webhook: checkout.session.completed
  • Generar link de descarga temporal (signed URL)
  • Página /checkout/success con descarga
Licensing
  • License type definitions (TypeScript)
  • PDF de licencia auto-generado (plantilla)
  • Guardar orden en Firestore
  • Email de confirmación con licencia + link
  • Protección: signed URLs expirables para audio
Stripe API Cloud Functions v2 Firebase Auth Firestore Resend
Fase 3 — Admin + Growth ~2 semanas
Dashboard de productor + herramientas de crecimiento
Un admin real donde puedas subir beats, ver ventas, trackear qué funciona. Más herramientas para que la gente comparta y vuelva.
Admin Dashboard
  • Dashboard con stats: ventas, revenue, top beats
  • CRUD de beats con drag & drop para audio
  • Upload de stems (archivos múltiples)
  • Gestión de precios por tier
  • Lista de órdenes con estados
  • Gráfico de ventas (chart.js o similar)
Growth Tools
  • PWA: service worker + manifest + install prompt
  • Share cards: link preview con waveform
  • Email list / newsletter signup
  • "Beats similares" recomendación
  • Playlists temáticas (curated sets)
Chart.js Service Worker Web Push API
Fase 4 — Polish & Scale ongoing
Detalles finos, testing, y escalar
Testing completo, performance tuning, i18n, y los detalles que separan algo bueno de algo profesional.
Testing
  • Vitest: stores, utils, firebase functions
  • Playwright: flujo de compra e2e
  • Lighthouse audit → score 90+
Polish
  • i18n: español + inglés
  • Error boundaries + fallback UI
  • Loading states con skeleton screens
  • Accessibility audit (a11y)
  • OG image generation dinámica
  • Rate limiting en Cloud Functions
  • Security rules audit
Vitest Playwright Paraglide.js Lighthouse CI
06 — Primeros Pasos
¿Qué hacemos hoy?
Si ya estás listo, arrancamos por acá. Esto es lo que necesita pasar para que la Fase 0 quede lista.
1
Decidí el stack
Arriba tenés las decisiones clave. Clickeá las opciones que preferís. Eso define todo lo que viene.
2
Armá el repo store
Init SvelteKit en el repo store, adapter-cloudflare, y deploy a Pages. En una tarde tenés el skeleton andando.
3
Migrar el diseño
Los colores, tipografía y estética del catalog ya están buenos. Los pasamos a componentes Svelte con CSS scoped. No reinventar — migrar.
4
Player + BeatCard
Los dos componentes más importantes. Si el player se siente bien y las cards se ven bien, todo lo demás es iteración.
