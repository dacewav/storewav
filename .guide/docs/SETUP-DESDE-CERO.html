SETUP DESDE CERO — DACEWAV.STORE

No necesitás saber código. Solo seguir pasos.

1
Instalar Visual Studio Code
El programa donde vas a escribir y editar código. Es gratis y es lo que usan los profesionales.
~5 minutos
¿Qué es esto?

VS Code es como un Bloc de Notas, pero hecho para programar. Te muestra los colores del código, te avisa si hay errores, y tiene una terminal integrada (la consola donde escribís comandos).

Paso 1.1 — Descargar

Andá a code.visualstudio.com y bajá la versión para tu sistema.

Paso 1.2 — Instalar

Abrí el archivo .zip descargado, arrastrá Visual Studio Code a tu carpeta Aplicaciones. Listo.

Abrí el .exe descargado, seguí el instalador. En una de las pantallas te pregunta "Add to PATH" — marcalo. Todo lo demás dejalo por defecto.

⚠ "Add to PATH" es importante. Si no lo marcás, después la terminal no va a encontrar los comandos. Si ya lo instalaste sin esto, reinstalá.
Paso 1.3 — Abrir VS Code por primera vez

Abrite VS Code. Vas a ver una pantalla con un menú a la izquierda y un área de "Bienvenida".

Cerrá la pestaña de bienvenida si querés. Ya está.

✓ Si VS Code se abrió bien, este paso está completo.
2
Instalar Node.js
El motor que hace funcionar todo por detrás. Sin esto no podés crear ni correr el proyecto.
~5 minutos
¿Qué es esto?

Node.js es un programa que permite que tu computadora entienda JavaScript (el lenguaje en el que está hecho tu proyecto). Cuando alguien dice "corré npm install", necesita Node.js instalado.

Paso 2.1 — Descargar

Andá a nodejs.org. Te va a mostrar dos versiones:

Elegí la que dice "LTS" (Long Term Support) — es la versión estable. La otra dice "Current" y es para experimentar.

Paso 2.2 — Instalar

Abrí el archivo .pkg descargado y seguí los pasos. Todo por defecto, siguiente-siguiente-siguiente.

Abrí el .exe y seguí el instalador. De nuevo: si te pregunta por PATH, asegurate de que esté marcado. Todo lo demás por defecto.

Paso 2.3 — Verificar que funciona

Ahora necesitamos abrir la terminal (la consola de comandos). Es como una chat con tu computadora donde le escribís instrucciones.

Cómo abrir la terminal en Mac:

Presioná ⌘ + Espacio, escribí "Terminal", y dale Enter. Se abre una ventanita negra (o blanca) con un cursor parpadeante.

Cómo abrir la terminal en Windows:

Click derecho en el botón de Windows (abajo a la izquierda) → elegí "Terminal" o "PowerShell". También podés buscar "terminal" en el menú de inicio.

En la terminal, escribí esto y dale Enter:

node --version

Si te muestra algo como v22.x.x o v20.x.x, Node.js está instalado correctamente.

💡 La terminal se va a volver tu mejor amiga. Es donde vas a correr comandos como "npm run dev" para ver tu proyecto andando. No tengas miedo — no se va a romper nada.
3
Instalar Git + crear cuenta GitHub
Git guarda el historial de tu código. GitHub es donde vive tu proyecto en la nube.
~10 minutos
¿Qué es Git?

Imaginá que cada vez que guardás tu proyecto, Git saca una "foto" de cómo está todo. Si algo se rompe, podés volver a cualquier foto anterior. Es como tener Ctrl+Z infinito para todo tu proyecto.

¿Qué es GitHub?

GitHub es como Google Drive pero para código. Ahí se guarda tu proyecto, y cuando lo conectás a Cloudflare, cada vez que subís un cambio, tu tienda se actualiza automáticamente.

Paso 3.1 — Instalar Git

Mac ya trae Git instalado (o te pide instalarlo al abrir la terminal por primera vez). Abrí la terminal y escribí:

git --version

Si te muestra una versión, ya lo tenés. Si te pide instalar "Command Line Tools", aceptá.

Andá a git-scm.com y bajá el instalador. Todo por defecto.

Paso 3.2 — Crear cuenta en GitHub

Andá a github.com y creá una cuenta (si ya tenés una, usala).

Nombre de usuario sugerido: dacewav

Paso 3.3 — Crear el repositorio "store"

En GitHub, click en el + arriba a la derecha → "New repository"

Configurá así:
Repository name: store
Description: Professional beat store
Visibility: Private
Initialize with: NO marcar nada

→ Click "Create repository"
⚠ NO marques "Add a README file", ".gitignore", ni "license". Tiene que estar vacío. Lo vamos a llenar desde tu computadora.
4
Crear el proyecto SvelteKit
Acá nace tu tienda. Un comando crea toda la estructura base del proyecto.
~5 minutos
Paso 4.1 — Abrir la terminal en VS Code

Abrí VS Code. Andá al menú de arriba: Terminal → New Terminal. Se abre una terminal dentro de VS Code (abajo).

Atajo: ⌘ + ` (la tecla de acento, arriba del Tab)

Atajo: Ctrl + ` (la tecla de acento, arriba del Tab)

Paso 4.2 — Navegar a donde querés tu proyecto

Escribí esto para ir a tu Escritorio (o donde quieras):

cd ~/Desktop
cd $HOME\Desktop
Paso 4.3 — Crear el proyecto

Escribí este comando (copia y pega):

npm create svelte@latest store

Te va a hacer unas preguntas. Elegí así:

Which Svelte app template?
→ Skeleton project (Enter)

Add type checking with TypeScript?
→ Yes, using TypeScript syntax (Enter)

Select additional options:
→ ESLint (Enter para marcar)
→ Prettier (Enter para marcar)
→ Enter para confirmar
Paso 4.4 — Instalar las dependencias

Entrá a la carpeta del proyecto e instalá todo lo que necesita:

cd store
npm install

Esperá a que termine (puede tardar 1-2 minutos la primera vez).

Paso 4.5 — Verificar que funciona
npm run dev

Te va a mostrar algo como:

VITE v5.x.x ready in xxx ms

➜ Local: http://localhost:5173/
➜ Network: ...

Abrí tu navegador y andá a http://localhost:5173. Si ves una página con "SvelteKit", ¡funciona!

🎉 ¡Tu proyecto ya existe! Es una página en blanco por ahora, pero es TU página en blanco. Todo lo que construyamos va a vivir acá.
5
Subir el proyecto a GitHub
Conectar tu computadora con GitHub para que tu código esté respaldado en la nube.
~5 minutos
¿Qué estamos haciendo?

Ahora el proyecto vive solo en tu computadora. Lo vamos a "subir" a GitHub para que:

1. Tengas un respaldo en la nube
2. Cloudflare pueda leer tu código y publicar tu tienda
3. Puedas trabajar desde cualquier computadora

Paso 5.1 — Inicializar Git en tu proyecto

En la terminal de VS Code (que ya debería estar en la carpeta store), escribí:

git init
git add .
git commit -m "initial sveltekit setup"

Esto significa: "Git, empezá a vigilar esta carpeta. Agregá todos los archivos. Y guardá esta foto como 'setup inicial'."

Paso 5.2 — Conectar con tu repo de GitHub

Andá a tu repo en GitHub (github.com/dacewav/store). Te va a mostrar unos comandos. Copiá los que dicen "...or push an existing repository".

Van a ser algo así (copiá los de TU pantalla, los míos son ejemplo):

git remote add origin https://github.com/dacewav/store.git
git branch -M main
git push -u origin main

Si te pide autenticación, GitHub te va a abrir el navegador para que te loguees.

Paso 5.3 — Verificar

Refrescá tu repo en GitHub. Si ves una lista de archivos como package.json, src/, svelte.config.js... ¡todo bien!

7
Crear proyecto en Firebase
Firebase es donde se guarda todo: los beats, los usuarios, las órdenes. Es gratis para empezar.
~10 minutos
¿Qué es Firebase?

Firebase es un servicio de Google que te da base de datos, autenticación (login), y hosting. Para tu tienda:

• RTDB = donde se guardan los beats (títulos, BPM, precios, links de audio)
• Firestore = donde se guardan las órdenes de compra y datos de usuarios
• Auth = el sistema de login con Google

Paso 7.1 — Crear cuenta

Andá a console.firebase.google.com con tu cuenta de Google (daceidk@gmail.com).

Paso 7.2 — Crear proyecto

Click en "Create a project" (Crear un proyecto).

Project name: dacewav-store

Google Analytics: Podés desactivarlo por ahora

→ Click "Create project"
→ Esperá unos segundos → "Continue"
Paso 7.3 — Activar Realtime Database

En el menú de la izquierda: Build → Realtime Database → Create Database

Elegí la región más cercana
Para las rules, elegí: "Start in test mode"
(después lo protegemos con las reglas de seguridad)
Paso 7.4 — Activar Firestore

Build → Firestore Database → Create Database

Región: la misma que elegiste antes
Rules: "Start in test mode"
Paso 7.5 — Activar Authentication

Build → Authentication → Get started

En la pestaña "Sign-in method", habilitá:

Google → Enable → Email: daceidk@gmail.com → Save

Crear cuenta en Cloudflare
Cloudflare es donde se publica tu tienda en internet. Gratis y rápido.
~5 minutos
¿Qué es Cloudflare?

Cloudflare es el servicio que publica tu tienda en internet. Cuando alguien escribe dacewav.store en su navegador, Cloudflare les muestra tu página. Es gratis para empezar.

Paso 8.1 — Crear cuenta

Andá a dash.cloudflare.com/sign-up y creá una cuenta con tu email.

Paso 8.2 — Crear proyecto en Pages

En el menú de la izquierda: Workers & Pages → Create → Pages → Connect to Git

Conectá tu cuenta de GitHub
Seleccioná el repo: dacewav/store

Build settings:
Framework preset: SvelteKit
Build command: npm run build
Build output directory: .svelte-kit/cloudflare

→ Click "Save and Deploy"
💡 El primer deploy va a fallar porque todavía no configuramos el adapter de Cloudflare. No te preocupes, lo arreglamos en el siguiente bloque.
9
Crear cuenta en Stripe
Stripe es el sistema de pagos. Cuando alguien compre un beat, Stripe procesa el dinero.
~10 minutos
¿Qué es Stripe?

Stripe es el servicio que procesa pagos con tarjeta de crédito/débito. Cuando un artista quiera comprar tu beat, Stripe se encarga de cobrarle y depositarte el dinero.

Paso 9.1 — Crear cuenta

Andá a stripe.com → "Start now"

Creá tu cuenta con tu email. Te van a pedir datos personales y bancarios para cuando quieras cobrar.

Paso 9.2 — Obtener las API keys

En el dashboard de Stripe: Developers → API keys

Vas a ver dos claves:

Publishable key: pk_test_...
Secret key: sk_test_... (click "Reveal")

Guardá las dos. La "secret key" es secreta — nunca la compartas.

Por ahora estamos en "test mode" (modo de prueba). Cuando estés listo para cobrar de verdad, activás "live mode".

💡 Las claves de test empiezan con pk_test_ y sk_test_. Las de producción empiezan con pk_live_ y sk_live_. Por ahora usamos las de test.

Instalar .guide/ en tu proyecto
El sistema de documentación que nos mantiene organizados. Ya lo tenemos armado, solo hay meterlo en el repo.
~3 minutos
¿Qué es esto?

El sistema .guide/ son 11 archivos de documentación que le dan contexto completo a cualquier persona (o IA) que trabaje en tu proyecto. Sin esto, cada vez que arrancás una sesión nueva, hay que explicar todo de vuelta.

Paso 10.1 — Crear la carpeta

En VS Code, en el panel de la izquierda (explorador de archivos), hacé click derecho en la raíz del proyecto → "New Folder" → escribí .guide

Paso 10.2 — Crear los archivos

Dentro de .guide/, creá estos archivos (click derecho → New File):

00-OVERVIEW.md
01-ARCHITECTURE.md
02-DATA-MODELS.md
03-AUTH-ADMIN.md
04-LICENSING.md
05-ROADMAP.md
06-CONVENTIONS.md
07-DEPLOYMENT.md
08-DECISIONS-LOG.md
09-CONTEXT-BRIEF.md
10-CHANGELOG.md
Paso 10.3 — Pegar el contenido

Cada archivo lo generamos en la sesión anterior. Para no hacer esto de a uno, te voy a dar un solo comando que crea todo.

En la terminal de VS Code, escribí:

mkdir -p .guide

Después, abrí cada archivo en VS Code y pegá el contenido del generador .guide que hicimos antes. O si preferís, cuando arranquemos a construir yo te los voy generando directamente.

💡 El contenido de cada archivo ya lo tenemos. En la próxima sesión, yo te los voy a ir generando directamente en tu proyecto. No tenés que copiarlos manualmente.
